# RFO UI Emulator — Project Instructions

## Overview
Interactive web-based emulator for RF Online interface concepts. No backend, no build step — just HTML/CSS/JS served from a static host (GitHub Pages).

## Architecture
```
index.html                ← Entry point, viewport, control panel
css/core.css              ← Global styles, CSS variables, panel theme
js/app.js                 ← Bootstrap: loads manifest, inits managers
js/core/
  settings.js             ← Global state (scale, bounds, mode), localStorage
  window-manager.js       ← Window registry, open/close/focus, z-index stack
  drag-engine.js          ← Pointer-event drag with screen bounds
  resize-engine.js        ← Resize handles, min/max constraints
  context-menu.js         ← Custom right-click menu
  layout-manager.js       ← Preset save/load, URL sharing, auto-save
  comment-manager.js      ← Pin annotations on windows
  export-manager.js       ← html2canvas PNG export, batch ZIP
windows/
  manifest.json           ← Registry of all windows
  _template/              ← Boilerplate for new windows (see WINDOW.md)
  {window-id}/            ← Each window: config.js, template.html, style.css
presets/                  ← Built-in preset files
assets/backgrounds/       ← Background images/videos
```

## How to Create a New Window
1. Copy `windows/_template/` → `windows/{your-window-id}/`
2. Edit the 3 files: `config.js`, `template.html`, `style.css`
3. Add an entry to `windows/manifest.json`
4. See `windows/_template/WINDOW.md` for full documentation

## CSS Variables Reference (defined in core.css :root)
- `--ui-scale` — Current scale factor (1.0 = 100%)
- `--viewport-w`, `--viewport-h` — Base viewport dimensions (1920, 1080)
- `--z-background`, `--z-windows`, `--z-overlays`, `--z-context-menu`, `--z-panel`
- `--panel-bg`, `--panel-border`, `--panel-accent`, `--panel-text`

## Core APIs Available to Windows

### WindowManager (`js/core/window-manager.js`)
```js
import { windowManager } from '../../js/core/window-manager.js';
windowManager.open(id)      // Open window
windowManager.close(id)     // Close window
windowManager.toggle(id)    // Toggle open/close
windowManager.focus(id)     // Bring to front
```

### Settings (`js/core/settings.js`)
```js
import { settings } from '../../js/core/settings.js';
settings.get('scale')       // Current scale factor
settings.get('mode')        // 'design' | 'export' | 'comment'
settings.on('mode', fn)     // Subscribe to changes
```

## Key Decisions
- **No Shadow DOM**: CSS scoped via `[data-window-id]` attribute selectors for html2canvas compatibility
- **No framework**: Vanilla JS + ES Modules for maximum simplicity
- **Base resolution**: 1920×1080, scaled via CSS transform
- **Export format**: PNG with transparency via html2canvas
- **Comments**: Pin annotations stored in presets, shared via JSON/URL

## Import Windows via GitHub URL
You can import a custom window module directly from a GitHub repository via the control panel by providing a branch URL in the format:
`https://github.com/{user}/{repo}/tree/{branch}`

This dynamically loads `config.js`, `template.html`, and `style.css` without requiring local Git changes.

## Export Clipping / Image Cuts Mitigation
To ensure `html2canvas` does not cut off box-shadows, glows, or thick borders during PNG exports, **the export manager automatically applies temporary padding and negative margins** to any element marked with `data-export` just before capturing it, and removes it immediately after.

**Correct Implementation:**
You do **not** need to manually wrap elements in padded `<div>` containers. Doing so will break the application's CSS Grid and Flexbox layouts. Simply attach the `data-export` attribute directly to the element you want to capture, and the system handles the rest.

```html
<!-- Simply attach the attribute to the element -->
<div class="my-element" data-export="my-element">
  ... content with box-shadows ...
</div>
```

# How to Create a New Window — Guide for Developers & AI Agents

## Overview

Each window in the RFO UI Emulator is an independent module consisting of 3 files inside its own folder under `windows/`:

```
windows/
  my-window/
    config.js       ← Module config (position, drag handle, resize, export definitions)
    template.html   ← HTML content of the window
    style.css       ← Window-specific styles (auto-scoped by core)
```

## Step-by-Step: Creating a New Window

### 1. Copy the Template

Copy the entire `windows/_template/` folder and rename it to your window's ID:

```
windows/_template/  →  windows/inventory/
```

The folder name **must match** the window's `id` in config.js.

### 2. Register in Registry

Add your window's ID to `windows/registry.json`:

```json
[
  "player-status",
  "inventory"
]
```

### 3. Edit config.js

```js
export default {
  id: 'inventory',
  title: 'Inventory',
  defaultPosition: { x: 100, y: 100, width: 400, height: 500 },
  defaultOpen: false,
  dragHandle: '.inventory-header',
  resizable: {
    enabled: true,
    handles: ['se'],
    minWidth: 300, minHeight: 400,
    maxWidth: 800, maxHeight: 900,
  },
  exports: [
    { selector: '[data-export="inventory-full"]', name: 'inventory-full', label: 'Full Inventory' },
  ],
  init(container) {
    // Optional interactive logic
  },
};
```

### 4. Edit template.html

Write your window's HTML. Important attributes:

```html
<div data-export="inventory-full" style="padding: 10px; margin: -10px;">
  <div class="inventory-container">
    <div class="inventory-header">
      <span>INVENTORY</span>
      <button class="close-btn">✕</button>
    </div>
  </div>
</div>
```

### 5. Edit style.css

Write plain CSS. The core loader automatically prefixes all selectors with `[data-window-id="inventory"]`.

```css
.inventory-container {
  width: 100%;
  height: 100%;
  background: rgba(10, 14, 24, 0.92);
}
```

## Key Concepts

### CSS Scoping Rules
- The loader prefixes ALL your CSS selectors with `[data-window-id="your-id"]`
- `:root` and `:host` selectors are replaced with the scope selector
- `@keyframes` and `@media` are passed through unmodified
- Use class names freely — they won't conflict with other windows
- **Avoid** using IDs in window CSS (use classes instead)
- If CSS variables fail to apply properly or cause rendering issues due to this scoping, inline the CSS values directly into the elements instead.
